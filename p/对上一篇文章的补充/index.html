对上一篇文章的补充

Aug 05, 2025
阅读时长: 3 分钟

在Discord中

作者Nanashi补充了部分问题的答案

现在继续补充

1.Chart Cyanvas 上的图表将要永远消失吗？

“不！我会尽力保留这些图表。”

2.我们是通过 tellonym 还是 Discord 获取我们的私人图表？

“在 Discord 上，特别是在一个私人线程中，在当前计划中。”

3.Chart Cyanvas 是什么时候制作的？如果这是愚蠢的问题，请原谅

“趣闻：虽然是在 sekai 三周年之前，但 cyanvas 有自己的痕迹笔记。”

4.你说过你可以提供一个服务，让人们可以通过一些二进制代码来托管自己的 cc 服务器（如果我没记错的话），那这个服务是如何运作的呢？

“Sonolus 服务器本身很简单；它只是一个 HTTP 服务器。

我在考虑做一个小型 cc 替代品：

转换图表，调整图像大小，重新编码音频

允许哪些人可以上传图表

GUI

不会有"官方"的服务器。

至少应该足够用来做同人作品。”

5.

能否下载私密图表？我在那里做了一些小工作：(

“我会尽力。”

6.非常感谢您多年来一直运行 CC。您制作的程序进行自托管是否有最低规格要求？

“在 1.0.0 之前，cyanvas 在 Oracle Cloud 的 A1 实例上运行，配备 2 个核心和 12GB 内存。但我认为你应该关注存储问题；我使用了 R2。”

7.你说的"public chart 的 scp"是什么意思？（比如，在这个语境中 scp 是什么意思？）

“SCP 是 Sonolus Collection Package 的缩写，我记得是这样。

简而言之，你可以将图表导入你的 Sonolus。这不应该花那么多钱。”

8.感谢您在这两年里为 Cyanvas 提供主机。这段时间里制图是我的主要爱好，但现在我想是时候继续前行了。问题：一旦您存档这些图表，是否可以下载？

“假设你在问"我可以在新的存档服务器上使用 usc 吗？”

回答：取决于设置。我在考虑尊重图表的设置，因为数据库本身并没有丢失。更新：对于那些不允许下载的图表，我会加密 usc。

当需要时，我会将密钥发送给我可以信任的人。"

9.

随着 Chart Cyanvas 的落幕和即将到来的自托管，这个 Discord 服务器能否专门用于上传人们可以下载的图表？也许可以设立一个论坛频道，让人们可以标记和上传图表。由于 Sonolus 上不会有一个官方的 Chart Cyanvas 服务器，我认为仍然能够在 Discord 服务器上分享图表，并有一个统一的上传、分享和游玩他人图表的地方会非常棒。

“这可能是一种方法，但实际上，我想与 sekai 同人社区保持距离。

你们中有些人可能知道：我是 VOICEVOX（一款 TTS 应用，例如你可以让 Zundamon 说话）的维护者之一。我想把更多资源投入其中。

所以我认为现在是结束的好时机。对此我感到抱歉。”

10.你为什么要做一个 Tellonym？

“我认为会有很多问题需要解答，但我不想跑题，所以提出一个。

我向 ChatGPT 询问"嘿你能找到 marshmallow-qa.com 的英文对应吗”，找到了 Tellonym"

总结：向伟大的Nanashi(无名氏)致敬！
